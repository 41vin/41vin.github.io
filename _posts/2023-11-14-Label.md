# Label

## 레이블을 이용한 파드 구성

실제 에플리케이션을 배포할 때, 파드 수가 증가함에 따라 파드를 부분 집합으로 분류할 필요가 있다.

 예를 들어 마이크로서비스 아키텍쳐의 경우 배포된 마이크로서비스의 수는 20개를 가볍게 초과한다. 이러한 구성 요소는 복제되어 여러 버전 혹은 릴리스가 동시에 실행된다. 이로 인해 시스템에 수백 개의 파드가 생길 수 있다. 파드를 정리하는 메커니즘이 없다면 난장판이 된다.

 모든 개발자들은 어떤 파드가 어떤 것인지 쉽게 알 수 있도록 임의의 기준에 따라 작은 그룹으로 조직하는 방법이 필요하다. 각 파드에 대해 개별적으로 작업을 수행하기보다 특정 그룹에 속한 모든 파드에 관해 한번에 작업하기를 원하고 있기 때문이다.

 레이블(Label)을 통해 파드와 기타 다른 쿠버네티스 오브젝트의 조직화가 이루어진다. 클러스터에 접속할 수 있는 개발자와 운영자는 파드 ㅔㄹ이블을 보고 시스템 구조와 각 파드가 적합한 위치에 있는지 볼 수 있다.



### 레이블 관련 커맨드

- 기존 파드 레이블 조회

```bash
$ k get po --show-labels

# 특정 레이블에만 관심이 있는 경우
$ k get po -L creation_method,env
```



- 기존 파드 레이블 수정

```bash
$ k label po <pod-name> <key>=<value>
```

> 기존 레이블을 변경할 때에는 --overwrite 옵션이 필요하다.

## 레이블 셀렉터를 이용한 파드 부분 집합 나열

레이블 셀렉터는 특정 값과 레이블을 갖는지 여부에 따라 리소스를 필터링하는 기준이 된다. 레이블 셀렉터는 리소스 중에서 다음 기준에 따라 리소스를 선택한다.

- 특정한 키를 포함하거나 포함하지 않는 레이블
- 특정한 키와 값을 가진 레이블
- 특정한 키를 갖고 있지만, 다른 값을 가진 레이블



### 레이블 셀렉터 관련 커맨드

- 레이블 셀렉터를 사용해 파드 나열

```bash
$ k get po -l creation_method=manual

NAME			READY			STATUS			RESTARTS			AGE
...				...				...				...					...
```



- 임의의 레이블을 가지고 있지만, 값은 무엇이든 상관없는 파드를 보려면 다음 명령을 실행한다.

```
$ k get po -l <label>
```



- 임의의 레이블을 가지고 있지 않은 파드를 조회하는 커맨드

```bash
$ k get po -l '!<label>'
```

> !<label>을 작은 따옴표로 감싸 Bash 셀이 느낌표를 처리하지 않도록 한다.

## 레이블과 셀렉터를 이용해 파드 스케쥴링 제한

 쿠버네티스의 전체적인 아이디어는 그 위에 실행되는 애플리케이션으로부터 실제 인프라스트럭처를 숨기는 것에 있기에 파드가 어떤 노드에 스케줄링 되어야 하는지 구체적으로 지정하지 않는다. 그로 인해 애플리케이션이 인프라스트럭쳐와 결합되기 떄문이다. 그러나 정확한 노드를 지정하는 대신 필요한 노드 요구 사항을 기술하고 쿠버네티스가 요구 사항을 만족하는 노드를 선택하도록 한다. 이는 **노드 레이블**과 **레이블 셀렉터**를 통해 할 수 있다.



### 워커 노드 분류에 레이블 사용

`gpu=true` 레이블을 노드 중 하나에 추가한다.

```bash
$ k label node node01 gpu=true
```



`gpu-true` 레이블을 가진 노드를 나열하려면 아래의 커맨드를 입력한다.

```bash
$ k get nodes -l gpu=true
```



### 특정 노드에 파드 스케쥴링

 GPU를 필요로 하는 새로운 파드를 배포해야 한다고 가정한다. 스케쥴러가 GPU를 제공하는 노드를 선택하도록 요청하려면, 파드의 YAML 파일에 노드 셀렉터를 추가해야 한다. 아래의 YAML 파일과 같다.

```yaml
apiVersion: v1
kind: Pod
...
...
spec:
  nodeSelector:
    gpu: "true"
  containers:
  - image: ...
  ...
```

 spec 섹션 안에 nodeSelector 필드를 추가하였다. 파드를 생성할 때, 스케쥴러는 gpu=true 레이블을 가지고 있는 노드 중에서 선택한다.
